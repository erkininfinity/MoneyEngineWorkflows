# OpenAI Codex for OSS Notes

This document keeps concise public notes for the Money Engine Workflows
maintainer workflow and the OpenAI Codex for OSS application.

---

## Official Form Requirements

The OpenAI form asks for:

- public GitHub username;
- public GitHub repository URL;
- maintainer role;
- why the repository qualifies;
- interest in Codex Security and/or API credits;
- OpenAI organization ID;
- how API credits will be used;
- optional additional notes.

## Suggested Answers

### Role

I am the primary maintainer and creator of Money Engine Workflows. I lead
workflow architecture, n8n template quality standards, CLI validation, safety
rules, documentation, releases, issue triage, and review of workflow
contributions.

### Why This Repository Qualifies

Money Engine Workflows is an OSS library of safe, human-in-the-loop n8n revenue
automation workflows, prompts, blueprints, examples, and validator tooling for
founders and small businesses, with explicit no-secrets and no-auto-spam rules.

### API Credit Usage

We will use API credits for workflow QA and OSS maintenance: prompt/eval
fixtures, sample output generation, manifest validation, issue triage, PR
review, docs, release notes, and regression checks for AI-powered workflow
templates.

### Additional Notes

The project turns practical AI automations into reusable OSS templates instead
of private one-off n8n exports. Codex would help maintain workflow quality,
review new templates, and keep safety rules consistent as the catalog grows.
